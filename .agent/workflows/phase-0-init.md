# Phase 0: プロジェクト初期化ワークフロー

## 目的・概要

このフェーズでは、新規プロジェクト開始時の**初期設定**を行う。Jiraプロジェクトの紐付け、Gitリポジトリの準備、README.mdの作成など、開発を始める前の土台を整える。

**重要**: このフェーズはArchCraftフレームワークを複製した直後、Phase 1（要件定義）の前に実行する。

## 前提条件

- [ ] ArchCraftフレームワークが複製済みであること
- [ ] Jira MCPが設定済みであること（オプション：Jira連携する場合）
- [ ] GitHub MCPまたはGitが使用可能であること

---

## ステップ 1: プロジェクト基本情報の確認

ユーザーに以下の基本情報を確認する：

> 「プロジェクト名は何にしますか？」
> 「このプロジェクトを一言で説明すると？」

確認後、`PRODUCT.md` を更新：
- **プロジェクト名:** を記入
- **概要:** を記入
- **開始日:** を今日の日付で記入

---

## ステップ 2: Jiraプロジェクトの確認・紐付け

### Jira MCPが利用可能な場合

1. Jira MCPを使用してアクセス可能なリソースを確認
   ```
   mcp_atlassian_getAccessibleAtlassianResources
   ```

2. 利用可能なプロジェクト一覧を取得
   ```
   mcp_atlassian_getVisibleJiraProjects
   ```

3. ユーザーに確認：
   > 「以下のJiraプロジェクトが見つかりました。どれを使用しますか？それとも新規作成しますか？」
   - 既存プロジェクトを選択 → プロジェクトキーをメモ
   - 新規作成 → Jira管理画面で作成してもらう

4. `PRODUCT.md` の `Jira プロジェクトキー:` を更新

### Jira MCPが利用できない場合

ユーザーに手動で確認：
> 「Jiraのプロジェクトキーを教えてください（例: PROJ-XXX）。Jiraを使わない場合は「なし」と答えてください」

---

## ステップ 3: Gitリポジトリの確認・作成

### 現在の状態を確認

```bash
git status
```

### パターン別対応

#### A) Gitリポジトリが存在しない場合

```bash
git init
git add .
git commit -m "chore: ArchCraftフレームワークで初期化"
```

ユーザーに確認：
> 「GitHubにリモートリポジトリを作成しますか？」

Yesの場合：
1. **GitHub MCPがある場合**: MCPでリポジトリ作成
2. **ない場合**: ユーザーにGitHubで手動作成してもらい、URLを教えてもらう

```bash
git remote add origin <repository-url>
git push -u origin main
```

#### B) Gitリポジトリが既に存在する場合

ユーザーに確認：
> 「既存のGitリポジトリが検出されました。このまま使用しますか？それとも新しいリポジトリを作成しますか？」

- 既存を使用 → 次のステップへ
- 新規作成 → 別ディレクトリへの移動を提案

## ステップ 4: README.md の初期作成

プロジェクトルートに `README.md` を作成する。

```markdown

{一言での説明}

## 概要

> ⚠️ このセクションは Phase 1（要件定義）完了後に更新されます

## 技術スタック

> ⚠️ このセクションは Phase 2（技術選定）完了後に更新されます

## セットアップ

> ⚠️ このセクションは Phase 5（環境構築）完了後に更新されます

## 開発フレームワーク

このプロジェクトは [ArchCraft](https://github.com/xxx/archcraft) フレームワークを使用しています。

---

© {年} {プロジェクト名}
```
